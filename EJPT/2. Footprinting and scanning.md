### Windows Recon: Nmap Host Discovery

Step 4: Discover alive hosts and ports

- Command:
    
    - `nmap -Pn demo.ine.local`
        

Scan a specific port (expect “filtered” if closed)

- Example: scan port 443
    
    - `nmap -Pn -p 443 demo.ine.local`
        

Discover running application/version on specific ports

- Port 80:
    
    - `nmap -Pn -sV -p 80 demo.ine.local`

### Scan the Server I
- Multiple ports with version detection:
    
    - `nmap demo.ine.local -p 6421,41288,55413 -sV`
        
### Scan the Server II
DNS / bind server and UDP discovery

- Determine if bind DNS (example port 177 here) is open with aggressive scan:
    
    - `nmap demo.ine.local -p 177 -A`
        
- UDP port range scan:
    
    - `nmap demo.ine.local -p 1-250 -sU`
        
- UDP with version detection on specific ports:
    
    - `nmap demo.ine.local -p 134,177,234 -sUV`
        
- UDP version detection plus discovery scripts on a single port:
    
    - `nmap demo.ine.local -p 134 -sUV --script=discovery`
        

TFTP on discovered UDP port

- Connect to TFTP service (example on port 134):
    
    - `tftp demo.ine.local 134`
        
### Scan the Server III

Full TCP/UDP scans with timing template

- Full TCP port scan with faster timing:
    
    - `nmap demo.ine.local -T4 -p-`
        
- Full UDP scan with faster timing:
    
    - `nmap demo.ine.local -T4 -sU`
        
- Focused aggressive UDP scan (example SNMP on 161):
    
    - `nmap demo.ine.local -T4 -sU -p 161 -A`
        
### Assessment Methodologies: Footprinting and Scanning CTF 1

HTTP, FTP, MySQL basic interaction

- Verbose HTTP request:
    
    - `curl -v http://target.ine.local`
        
- Stealth + version detection against target:
    
    - `nmap -sS -sV target.ine.local`
        
- FTP connection:
    
    - `ftp target.ine.local`
        
- MySQL connection:
    
    - `mysql -h <ip> -u db_admin -p`
        
- MySQL enumeration example:
    
    - `show databases;`
        

### Windows Recon: SMB Nmap Scripts
SMB enumeration on port 445  
Step 4: List supported SMB protocols and dialects

- Command:
    
    - `nmap -p445 --script smb-protocols demo.ine.local`
        

Step 5: SMB security mode enumeration

- Command:
    
    - `nmap -p445 --script smb-security-mode demo.ine.local`
        

Step 6: SMB session enumeration (with and without creds)

- Credentials: `administrator:smbserver_771`
    
- Without credentials:
    
    - `nmap -p445 --script smb-enum-sessions demo.ine.local`
        
- With credentials:
    
    - `nmap -p445 --script smb-enum-sessions --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 7: Enumerate all available SMB shares

- Command:
    
    - `nmap -p445 --script smb-enum-shares demo.ine.local`
        

Step 8: Enumerate Windows users

- Command:
    
    - `nmap -p445 --script smb-enum-users --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 9: Get SMB server statistics (uses 445 and 139)

- Command:
    
    - `nmap -p445 --script smb-server-stats --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 10: Enumerate available domains

- Command:
    
    - `nmap -p445 --script smb-enum-domains --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 11: Enumerate user groups

- Command:
    
    - `nmap -p445 --script smb-enum-groups --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 12: Enumerate services

- Command:
    
    - `nmap -p445 --script smb-enum-services --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
        

Step 13: Enumerate shares and list contents

- Command:
    
    - `nmap -p445 --script smb-enum-shares,smb-ls --script-args smbusername=administrator,smbpassword=smbserver_771 demo.ine.local`
